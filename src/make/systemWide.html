systemWide.mk

Intro

systemWide.mk holds globally available rules. THESE RULES ARE AVAILABLE IN ALL NODES AND LEAVES.

Rules

env

make env

List all the env variables - this includes all the $(MAKE) variables

m4command

make m4command

Spit out just the m4 command that will be run in this path

releaseVersion

make releaseVersion

Check the environment for a $(VERSION) variable. Confirms the value in it.

targetMap.mk

make targetMap.mk

create a file that gives a list of targets to build. This is useful in tool directories, because it uses the list of *.m4 files to generate the rules that are available. It creates a <filename> rule and a <filename>_debug rule. The debug rule will output the contents of the log file that results from running the rule

newComponent

make newComponent

When run in a directory below the tree, it will prompt for $NAME of the new component (the directory name) and the $TYPE of the component (currently listgen ONLY). It will create the appropriate depth and makefile files in the new location. Part of the makefile and depth.mk file generation process requires the user to decide if this new component is a node of a leaf. The main difference (currently) is that nodes can recurse their subnodes, leaves cannot. Leaves have access to generating documentation, nodes do not.
